System Overview

SPAKE2+ Protocol

  • Secure Password-Based Authenticated Key Exchange
  • Device and Vehicle mutual authentication
  • Cryptographic salt and key derivation
  • Evidence-based verification process

Secure Payload System

  • AES encryption with BouncyCastle provider
  • MAC (Message Authentication Code) chaining

Secure Communication Flow Architecture

1
SPAKE2+ Init
Protocol Setup & Key Generation
Protocol Parameters:
  • Scrypt parameters (cost, blockSize, parallelization)
  • Cryptographic salt generation
  • w0, w1 scalar derivation from password
  • Random scalar generation (x, y)
2
Point Exchange
Public Point Generation & Exchange
Exchange Process:
  • Device: X = x*G + w0*M
  • Vehicle: Y = y*G + w0*N
  • L = w1*G (shared computation)
3
Secret Computation
Key Derivation & Evidence Generation
Key Generation:
  • Z = x*(Y - w0*N) = y*(X - w0*M)
  • V = w1*(Y - w0*N) = y*L
  • K = SHA-256(X || Y || Z || V || w0)
  • CK, SK = Split(K)
4
Evidence Verification
Mutual Authentication
Evidence Exchange:
  • K1, K2 = HKDF(CK, "ConfirmationKeys")
  • Vehicle Evidence = CMAC(K1, Y)
  • Device Evidence = CMAC(K2, X)
  • Mutual Verification
5
System Keys
Final Key Generation
Derived Keys:
  • Kenc: Encryption Key
  • Kmac: Command MAC Key
  • Krmac: Response MAC Key
  • Long-term Shared Secret
6
Secure Messaging
Encrypted Communication
Message Security:
  • AES-CBC Encryption with Counter IV
  • CMAC Generation with Chaining
  • Sequential Counter (01h to FFh)
  • ISO/IEC 9797-1 Padding

Technical Specifications

SPAKE2+ Protocol

Cryptographic Parameters

  • Curve: NIST P-256 (secp256r1)
  • Key Size: 256-bit
  • Hash Function: SHA-256
  • MAC Algorithm: CMAC-AES-128
  • KDF: HKDF with SHA-256

Protocol Implementation

  • Password Generation and Processing
  • Public Point Generation (Device & Vehicle)
  • Shared Secret Computation
  • Key Derivation and Evidence

Payload Encryption

Encryption Parameters

  • Algorithm: AES-CBC
  • Key Size: 128-bit
  • MAC: CMAC-AES-128 (8-byte)
  • Padding: ISO/IEC 9797-1 method 2
  • Counter Range: 1-255

Security Considerations

  • Protection against replay attacks
  • Secure key exchange protocol
  • Message integrity and authenticity
  • Proper error handling
